# Contributing

## Arbeitsregeln

- Kleine, klare Commits bevorzugen.
- Neue Experimente zuerst in `notebooks/` erkunden.
- Wiederverwendbaren Code nach `src/` verschieben.
- Datenquellen und Annahmen dokumentieren.
- Tests ergaenzen, wenn Funktionen in `src/` erweitert werden.

## Branch-Namen

Beispiele:

- `feature/regression-baseline`
- `feature/model-selection`
- `docs/theory-notes`
- `fix/data-loader`
